# SpaceLens 建筑空间绩效评价平台

SpaceLens 是一款面向建筑空间使用后评价的本地分析软件，用于把平面图、人员定位、行为记录、环境监测和问卷数据转化为可视化图表、评价指标和可导出的分析结果。

软件适合建筑空间研究、空间运营评估、教学科研和方案复盘场景。所有数据默认在本机处理和保存。

## 主要能力

- 定位分析：到访频次、空间使用时长、移动速率、空间停留时长、空间聚类、人员密度、空间开放程度、拓扑连接关系、轨迹差异系数、轨迹长度。
- 环境分析：温度、湿度、光照、风速、噪声的空间分布和测点统计。
- 行为分析：行为发生人次、行为持续时长、行为平均发生率、行为复合程度、空间功能利用率。
- 满意度分析：整体满意度、空间单元满意度、设计要素满意度。
- 项目管理：历史项目保存、结果恢复、项目重命名、删除、导出。
- 项目对比：选择两个历史项目，对关键图表和统计结果进行并排对比。
- 结果导出：支持图片、Excel 和 ZIP 打包导出。Excel 包含图表背后的统计数据和热力图/插值矩阵等底层数据。

## 快速开始

### 启动软件

推荐使用桌面端：

```bash
python desktop_app.py
```

也可以启动 Web 服务：

```bash
python app.py
```

然后在浏览器访问：

```text
http://127.0.0.1:8080
```

桌面端支持原生文件选择、保存对话框和打开源文件位置，体验更完整。

### 安装依赖

首次运行前需要准备 Python 环境：

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Windows PowerShell：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## 使用流程

1. 打开软件，进入新建项目。
2. 填写项目名称、建筑类型、楼层信息和数据采集日期。
3. 上传平面图和可用的数据文件。
4. 点击开始分析。
5. 在结果页查看各指标图表和统计卡片。
6. 可按需导出单个指标，或保存整个项目结果。
7. 后续可在历史项目中查看、恢复、对比或重新导出。

## 数据准备

### 平面图

- 支持 PNG、JPG 等常见图片格式。
- 平面图坐标应与定位、行为、环境数据中的 `X`、`Y` 坐标一致。
- 黑色墙体或不可通行区域会用于减少热力图扩散。

### 定位数据

常用文件名示例：`location.xlsx`

常用列：

| 列名 | 说明 |
| --- | --- |
| `UserID` | 人员编号 |
| `Timestamp` | 时间 |
| `X` | 横坐标 |
| `Y` | 纵坐标 |
| `t` | 记录持续时间或时间间隔 |
| `Region` | 空间单元编号 |

### 行为数据

常用文件名示例：`behavior.xlsx`

常用列：

| 列名 | 说明 |
| --- | --- |
| `UserID` | 人员编号 |
| `X` | 横坐标 |
| `Y` | 纵坐标 |
| `Region` | 空间单元编号 |
| `BehaviorNum` | 行为编号 |
| `behaviortype` | 行为名称 |
| `t` | 行为持续时间 |

行为数据中值为 `/` 的行为会被过滤，不参与图例和统计图显示。

### 环境数据

常用文件名示例：`environment.xlsx`

必须包含：

| 列名 | 说明 |
| --- | --- |
| `X` | 测点横坐标 |
| `Y` | 测点纵坐标 |
| `ParameterNum` | 环境参数编号 |
| `Value` | 测量值 |

`ParameterNum` 对应关系：

| 编号 | 参数 |
| --- | --- |
| `1` | 温度 |
| `2` | 湿度 |
| `3` | 光照 |
| `4` | 风速 |
| `5` | 噪声 |

### 问卷数据

一键分析支持三类问卷：

| 文件 | 用途 |
| --- | --- |
| 整体满意度问卷 | 计算整体满意度 |
| 空间单元满意度问卷 | 计算各空间单元满意度 |
| 设计要素满意度问卷 | 计算各设计要素满意度 |

### 区域坐标数据

常用文件名示例：`region_coordinates.xlsx`

区域坐标用于空间开放程度、功能利用率等需要面积或区域边界的指标。没有区域坐标时，部分指标会使用默认面积或跳过。

## 结果导出

结果页支持三种导出方式：

- 保存图片：导出当前指标的结果图。
- 保存 Excel：导出当前指标的摘要和图表背后数据。
- 保存 ZIP：打包导出图片和 Excel。

保存整个项目时，ZIP 通常包含：

```text
images/        指标结果图片
data/          指标 Excel 数据
summary.json   全量摘要
README.txt     导出说明
```

Excel 中会尽量保留当前图表背后的计算数据，例如热力图矩阵、环境插值矩阵、区域统计表、拓扑转移矩阵、行为矩阵和满意度原始评分。

## 历史项目与项目对比

计算完成后，项目会自动记录到历史项目中。你可以：

- 查看历史项目结果。
- 从已保存结果恢复项目。
- 重命名或删除项目。
- 导出历史项目。
- 选择两个历史项目进行对比。

项目对比页会按指标并排展示两个项目的图表。环境参数会拆分为温度、湿度、光照、风速、噪声分别对比。

## 数据保存位置

默认数据目录：

```text
~/.spacelens/
├── projects.db
└── results/
```

其中：

- `projects.db` 保存历史项目记录。
- `results/<session_id>/` 保存图表、摘要、元数据和导入文件副本。

可以通过环境变量修改数据目录：

```bash
SPACELENS_DATA_DIR=/path/to/data python desktop_app.py
```

## 常见问题

### 为什么有些指标显示跳过？

通常是对应数据文件缺失，或文件中缺少该指标需要的列。可以检查上传文件是否包含 README 中列出的字段。

### 为什么历史项目打不开？

历史项目依赖本地结果目录恢复。如果结果目录被删除、移动或权限受限，需要重新计算或手动指定结果文件夹。

### 为什么图表和平面图对不上？

请检查平面图像素坐标是否与 `X`、`Y` 数据坐标一致。坐标超出图像范围时，系统会尝试自动归一化，但最好在数据准备阶段保持一致。

### 纯 Web 模式和桌面端有什么区别？

纯 Web 模式可以分析和查看结果，但桌面端提供更完整的本地能力，例如原生保存对话框、源文件定位和本地应用封装。
