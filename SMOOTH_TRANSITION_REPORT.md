# 热力图背景色平滑过渡优化报告

## 📋 问题描述

用户反馈：0值背景色比有数据的区域边缘颜色还浅，导致过渡不自然。

**原有方案的问题**：
- 高斯密度热力图：使用 `cm(0.15)` 作为背景色（较浅的紫色）
- RBF插值热力图：使用 `cm(0.5)` 作为背景色（中值色）
- 结果：背景色比数据边缘颜色浅，视觉上不连续

## ✅ 优化方案

**核心思路**：计算有数据区域的边缘颜色，用这个颜色填充背景，实现平滑过渡。

### 实现逻辑

#### 1. 高斯密度热力图 (`_make_heatmap_overlay`)

```python
# 计算有数据区域的最小归一化值（边缘颜色）
DATA_THRESH = 0.01
data_mask = fill_mask & (density_norm > DATA_THRESH)

if np.any(data_mask):
    # 取有效数据区域的5分位数对应的颜色作为背景色
    min_norm_val = float(np.percentile(density_norm[data_mask], 5))
    base_color = np.array(cm(min_norm_val)[:3], dtype=float)
else:
    # 无有效数据时，使用 colormap 最小值色
    base_color = np.array(cm(0.0)[:3], dtype=float)
```

**关键点**：
- 使用 `np.percentile(..., 5)` 取5分位数，避免极端值
- 背景色 = 数据边缘的颜色，实现自然过渡

#### 2. RBF插值热力图 (`_make_rbf_overlay`)

```python
data_mask = fill_mask & np.isfinite(field)

if np.any(data_mask):
    # 取有效数据的5分位数对应的颜色作为背景色
    min_norm_val = float(np.percentile(field_norm[data_mask], 5))
    base_color = np.array(cm(min_norm_val)[:3], dtype=float)
else:
    # 无有效数据时，使用 colormap 中值色
    base_color = np.array(cm(0.5)[:3], dtype=float)
```

## 🎯 技术细节

### 为什么使用5分位数？

1. **避免极端值**：直接取最小值可能受噪声影响
2. **代表边缘**：5分位数能较好地代表数据边缘的颜色
3. **平滑过渡**：略高于最小值，确保背景到数据的渐变自然

### 为什么保持 alpha=0.30？

- 半透明混合，保留底图的结构线（墙体、门窗等）
- 既能看到热力色，又能看到平面图细节
- 平衡可读性和美观性
