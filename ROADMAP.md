# SpaceLens 开发 Roadmap

本文档记录 SpaceLens 后续开发计划。README 面向用户说明，本文件面向开发和维护。

## 2.0 稳定版状态

2.0 版重点稳定了完整工作流：

- 桌面端封面、启动和白屏问题修复。
- 一键分析、历史项目恢复、项目对比流程稳定。
- 环境参数拆分为温度、湿度、光照、风速、噪声独立对比。
- 图表显示优化，包括满意度图表、拓扑图、轨迹图、聚类图和对比页图片布局。
- 行为数据中的 `/` 会过滤，不进入图例和统计。
- 导出 Excel 包含图表背后数据，项目 ZIP 保存多张结果图。
- session 轮询瘦身，避免大矩阵和重复图片导致前端卡住。

## 近期优先级

### P0 稳定性

- 为 `GUI/` 示例数据建立一键回归脚本，覆盖 run_all、结果轮询、导出 ZIP 和历史恢复。
- 给环境参数、拓扑、满意度、行为过滤、项目对比增加自动化 smoke test。
- 检查直接运行 `python app.py` 的纯 Web 模式，补齐 `/api/ready` 或前端兼容逻辑。
- 为导出接口增加大文件压力测试，确认 ZIP/Excel 在桌面端保存稳定。

### P1 用户体验

- 增加导入数据预检：在计算前提示缺少哪些文件、哪些列、哪些指标会跳过。
- 在结果页显示每个指标的跳过原因，而不是只显示“数据不足”。
- 优化计算进度显示，从固定指标总数改为“本次可计算指标数”。
- 为项目对比页增加按指标类型筛选和快速定位。
- 增加导出完成后的结果目录打开按钮。

### P1 数据与指标

- 统一数据列名映射，支持用户在导入时匹配字段。
- 支持更多环境参数别名和单位显示。
- 优化热力图、环境插值和遮罩策略的参数配置。
- 增加区域坐标缺失时的明确降级说明。
- 为满意度问卷支持更灵活的列结构。

### P2 工程结构

- 拆分 `api/analysis.py`：
  - `metrics/`：各指标计算。
  - `sessions.py`：session 缓存、恢复、轮询 payload。
  - `exports.py`：ZIP、Excel、图片导出。
  - `projects.py`：历史项目 API。
  - `paths.py`：桌面端路径解析、本地数据目录。
- 统一一键分析和单指标分析的前端状态模型。
- 清理历史报告、示例 ZIP、构建产物和 MATLAB 原型的版本管理策略。
- 补充 `.gitignore`，减少构建输出和临时文件进入仓库。

### P2 打包发布

- 建立 macOS 和 Windows 打包检查清单。
- 为 PyInstaller 构建加入版本号、图标、签名和发布说明。
- 明确示例数据是否随安装包分发。
- 建立 release tag 和 changelog 生成流程。

## 建议测试清单

每次发布前至少执行：

```bash
python -m py_compile app.py desktop_app.py api/analysis.py api/db.py
```

使用 `GUI/` 示例数据验证：

- 新建项目并完成一键分析。
- 环境参数 5 个子指标均能完成或正确跳过。
- 行为数据 `/` 过滤生效。
- 结果页切换、导出图片、导出 Excel、导出 ZIP。
- 保存整个项目。
- 历史项目恢复。
- 两个历史项目对比。

## 已知技术债

- `api/analysis.py` 职责过重，后续维护成本高。
- 结果图、导出数据和前端展示数据仍共用同一 result dict，需要继续边界化。
- 当前指标对列名强约定，用户数据稍有差异就容易跳过。
- 桌面端与纯 Web 模式能力不完全一致。
- 管理员页面适合本地调试，不应作为生产级权限系统。
